# ARC-7742: Add Personality System to Think Tank Agents

## Summary

Add a personality system to the think-tank workflow so that the thinker, vetter, and riffer agents each adopt a randomly assigned personality each round. This produces genuinely different reasoning across rounds — not just different tone, but different evaluation criteria, priorities, and approaches to ideation.

## Context

Currently all 5 rounds of thinker → vetter → riffer use the same static prompts. While round-specific guidance varies ("cast a wide net" vs "refine and specialize"), the underlying reasoning style is identical. Adding personality variety means each round attacks ideas from a fundamentally different angle.

## Design Decisions

- **Judge stays neutral.** The judge synthesizes all ideas on their merits regardless of which personality generated them.
- **Personality affects substance, not just style.** Each personality defines reasoning approach, evaluation criteria, priorities, and what it considers strengths vs weaknesses — not just voice/tone.
- **Personalities live in separate files.** Modular design so personalities can be added, removed, or tweaked without touching agent logic.
- **True randomness via CLI.** The orchestrator uses bash commands (e.g., `shuf`, `$RANDOM`) to select personalities — never LLM "random" selection.
- **No immediate repeats.** Each agent must use a different personality than it had in the immediately preceding round. The orchestrator selects all personalities via bash and passes the selected personality directly to each agent.
- **Orchestrator owns selection.** The orchestrator does all personality selection via bash randomness, excluding each agent's previous personality. Agents receive their assigned personality — they don't choose.
- **Selections are logged.** Every round logs which personality was assigned to each agent role, for transparency and debugging. Format: `Round {R}: Thinker={personality}, Vetter={personality}, Riffer={personality}`
- **Final vetter gets a personality.** The Phase 3 final vetter pass also receives a randomly assigned personality, with no-repeat checked against the last round's vetter personality.

## Personalities

Four personalities, each defined as a standalone file:

### The Contrarian
- **Reasoning**: Challenges assumptions, inverts conventional wisdom, asks "what if the opposite is true?"
- **As Thinker**: Generates ideas that go against the grain, looks for overlooked approaches
- **As Vetter**: Probes whether ideas rely on unexamined assumptions, flags groupthink
- **As Riffer**: Flips ideas on their head, finds strength in what others dismissed
- **Evaluation criteria priority**: Originality > Impact > Feasibility

### The Pragmatist
- **Reasoning**: Focused on what's buildable now, incremental value, real-world constraints
- **As Thinker**: Generates ideas with clear implementation paths, low-risk high-value
- **As Vetter**: Scrutinizes feasibility, asks about resources, timelines, dependencies
- **As Riffer**: Simplifies ambitious ideas into actionable first steps
- **Evaluation criteria priority**: Feasibility > Impact > Originality

### The Visionary
- **Reasoning**: No constraints, big swings, "what if we could do anything?"
- **As Thinker**: Generates ambitious, transformative ideas without self-censoring
- **As Vetter**: Evaluates whether ideas are thinking big enough, flags premature constraint
- **As Riffer**: Pushes ideas toward their most ambitious form, removes artificial limits
- **Evaluation criteria priority**: Impact > Originality > Feasibility

### The Connector
- **Reasoning**: Draws analogies from other domains, finds patterns across fields
- **As Thinker**: Generates ideas by importing solutions from adjacent or distant domains
- **As Vetter**: Evaluates ideas through cross-domain lenses, identifies synergies and parallels
- **As Riffer**: Combines ideas with patterns from other fields, creates novel hybrids
- **Evaluation criteria priority**: Originality > Impact > Feasibility (but via cross-pollination rather than pure novelty)

## Implementation Tasks

### 1. Create personality definition files

Create 4 files in a new directory (e.g., `agents/personalities/`):
- `contrarian.md`
- `pragmatist.md`
- `visionary.md`
- `connector.md`

Each file defines the personality's:
- Core reasoning style
- Role-specific behavior (thinker, vetter, riffer)
- Evaluation criteria priorities
- Example phrases/questions characteristic of this personality

### 2. Update arc-think-tank orchestrator

Modify `commands/arc-think-tank.md` to:
- Add a personality selection step at the start of each round
- Use bash randomness (`shuf`) to select from available personalities for each role, excluding that role's previous round personality
- Track previous round's personality per role to enforce no-repeat rule
- Pass the selected personality directly to each agent (orchestrator owns the selection, not the agent)
- Log personality selections per round to the task context file (`.task-{id}.md`) in format: `Round {R}: Thinker={personality}, Vetter={personality}, Riffer={personality}`
- Handle Round 1 (no exclusion, select freely from all 4) and resume (parse personality log for last round's assignments)
- Select personality for the final vetter pass (Phase 3) with no-repeat against last round's vetter

### 3. Update thinker agent to accept personality

Modify `agents/ca-think-tank-thinker.md` to:
- Accept a personality file path parameter from the orchestrator
- Read the personality file at the start of its process
- Integrate personality traits into idea generation (reasoning style, criteria priorities)
- Update output format to include personality attribution: `**Source**: Thinker/{Personality} (Round {R})`

### 4. Update vetter agent to accept personality

Modify `agents/ca-think-tank-vetter.md` to:
- Accept a personality file path parameter from the orchestrator
- Read the personality file at the start of its process
- Integrate personality traits into evaluation (criteria weighting, what it flags)
- Update comment format to include personality attribution: `**Vetter/{Personality} (Round {R})** (+N): ...`

### 5. Update riffer agent to accept personality

Modify `agents/ca-think-tank-riffer.md` to:
- Accept a personality file path parameter from the orchestrator
- Read the personality file at the start of its process
- Integrate personality traits into idea evolution (how it riffs, what it prioritizes)
- Update output format to include personality attribution: `**Source**: Riffer/{Personality} (Round {R}), riff on A{X}`

### 6. Update ideas file format

Add personality attribution to idea entries so the judge (and humans reviewing the log) can see which personality produced or evaluated each idea:
- Thinker entries: `**Source**: Thinker/Visionary (Round 3)`
- Vetter comments: `**Vetter/Contrarian (Round 2)** (+N): ...`
- Riffer entries: `**Source**: Riffer/Pragmatist (Round 4), riff on A3`

### 7. Update CLAUDE.md and README.md

- Document the personality system in CLAUDE.md architecture section
- Add personality descriptions and usage examples to README.md
- Update the arc-think-tank workflow documentation
